What are Datatypes?

Data exists in various formats such as words, numbers, date & time etc., Data types are used to define the kind of data that can be stored in a particular Location.

( or )

Simply, data types explain what kind of data can be stored.

Data is available in various formats and is classified into different types based on its nature.

v1.2-char

Char - fixed

It accepts ' A-Z ' , ' a-z ' , ' 0-9 ' , Special characters ( such as &, %, $, ?... ) and as well as combinations of charaters and numbers, with maximum length of up to 255 characters.

Syntax :

CHAR( n )

n : It specifies the maximum length of the string, where n can range from 0 to 255. If the string exceeds this limit, it will be truncated to fit within the specified limit.

  • CHAR() can accept numeric digits but they store them as characters ( text ), not as numbers.
  • The data enclosed by single quotes ( '  ' ).
  • It follows Fixed length Memory Allocation - This can lead to wasted storage space when the stored data is shorter than the defined length.

Let's take an example,

If you define CHAR(10) and store ' Hello ', it uses 5 characters to fill and remaining 5 with spaces, resulting in 'Hello_____'.

CHAR( 10 )

Cons

  1. It wastes storage space due to fixed-length allocation, as unused characters are filled with spaces.
  2. It has a maximum limit of 255 characters, so it cannot store data beyond this length.
v1.2-varchar

Varchar - variable

It accepts ' A-Z ' , ' a-z ' , ' 0-9 ' , Special characters ( such as &, %, $, ?... ) and as well as combinations of charaters and numbers, with maximum length of up to 65535 characters.

Syntax :

VARCHAR( n )

n : It specifies the maximum length of the string, where n can range from 0 to 65535.

  • Same like CHAR, It accepts numeric digits but they store them as characters.
  • The data enclosed by single quotes ( '  ' ).
  • It follows Variable length Memory Allocation - we can also retreive unused storage space when the data is shorter than the defined length.

Let's take an example,

If you define VARCHAR(7) and store ' MySQL ', it uses 5 characters to fill and remaining 3 with spaces, resulting in 'MySQL__'.

VARCHAR( 7 )

Pros

  1. Storage space wastage is avoided by VARCHAR(), as it uses variable-length allocation and stores only the actual data, the remaining space can be utilized for storing other data.
  2. The length limit in VARCHAR() is more flexible, with a max size of up 65535 characters, allowing it to store much larger data.

v1.2-Date

Date - Calender

Syntax :

DATE

DD-Mon-YYYY represents the Oracle date format, While YYYY-MM-DD follows the ANSI standard format.

  DD denotes the day ( 01 to 31 ).
    MM denotes the month in numeric form ( 01 to 12 )
    Mon represents the month in abbreviated text ( Jan to Dec )
    and YYYY indicates the year ( 1900 to 2026 ).

Binary Large Object

It is used to store large amount of binary data such as images, videos, audio files etc., with a maximum size of up to 4GB.

Syntax :

BLOB
