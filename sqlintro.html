What is SQL?

Structured Query Language ( SQL ) is the standard programming language used to manage, manipulate and retrieve data which is stored in relational databases. Precisely, SQL language is used for interacting with entities ( tables ) & their attributes ( columns ) and tuples ( rows ) with data values in relational database by using simple commands like CREATE, SELECT, ALTER and DROP to handle large amounts of data efficiently.

To understand SQL, we have to know about the basics terms like Data Database, DBMS and RDBMS. How they these are related to each other ? What are the limitations and Features ?

v1.1-db

Database - Storage unit

Database is an organized collection of structured data stored electronically in a computer system & managed by DBMS.

( or )

A repository of data - A program that stores data.

A Database is more than just storage - It is a structured, secure, scalable system that transforms raw data into usable information.
Structured : Data is organized in a clear format, easy to store, manage and query.
Secure : Protects data from from unauthorised access by authentication and encryption.
scalable : Handle large amount of data. Data volume increases along with user demand without losing efficiency.

v1.1-dbms

DBMS - Manager

DataBase Management System shortly as DBMS, which is used to maintain the database, like a store manager who stocks shelves and helps you to find items easily & rapidly.

A software that manages data in a structured way, often using files ( or ) hierarchical models.
To communicate with DBMS software, we use QL ( Query Language ).

Drawback

Stores data in file format, meaning each file is dedicated to specific topic, so unused space in those files cannot be utilized for other topics. This leads to wasted storage and can slow down overall database performance.

v1.1-rdbms

RDBMS - Organised Layout

Relational Database Management System, a type of DBMS that organises data in tables ( relations ) with rows and columns. It enforces relationships using primary keys & foreign keys.

To communicate with RDBMS, we use SQL ( Structured Query Language ).

The main reason to introduce the RDBMS is, to overcome the storage issues associated with earlier database systems. So, RDBMS ensures data integrity, reduces redundancy and enables efficient querying and updating of data.
Data integrity : Uniqueness of data.
Reduces redundancy : Minimise the repetation of data, by splitting into more tables.
